# K8s & DnD 5e

Русскоязычная учебная кампания, которая объясняет Kubernetes через приключение в духе D&D 5e. Вместо сухого списка ресурсов здесь есть мир, фракции, квесты, боссы и артефакты, но рядом с каждой метафорой всегда стоит прямое техническое объяснение.

## Для кого это

- Для новичков, которые хотят понять Kubernetes с нуля.
- Для разработчиков и DevOps-инженеров, которым легче учиться через сюжет и практику.
- Для тех, кто хочет не просто читать про `Pod` и `Service`, а руками поднимать объекты в локальном кластере.

## Что внутри

- `chapters/` - главы кампании.
- `labs/` - практические квесты по каждой теме.
- `manifests/` - готовые YAML-манифесты.
- `cluster/` - конфигурация локального кластера `kind`.
- `demo-app/` - маленькое учебное приложение для финального рейда.
- `glossary/` - словарь терминов и соответствий D&D <-> Kubernetes.
- `assets/` - описание мира и фракций.

## Как проходить кампанию

1. Прочитайте [Пролог](./chapters/00-prologue.md).
2. Поднимите локальный кластер по [лабу настройки](./labs/00-setup-kind.md).
3. Идите по главам по порядку.
4. После каждой главы выполните соответствующий лаб.
5. В финале соберите образ локального приложения и разверните его в `kind`.

## Prerequisites

- `Docker`
- `kubectl`
- `kind`
- базовое понимание терминала

Официальные команды из документации `kind`:

```bash
go install sigs.k8s.io/kind@v0.30.0
kind create cluster --config cluster/kind-config.yaml --name k8s-dnd
kubectl cluster-info
```

Если `kind` уже установлен другим способом, этого достаточно.

## Сюжетный каркас

В мире Кубернесии есть архипелаг узлов, над которым следит Совет Контрольного Плана. Игроку предстоит провести отряд через восемь испытаний: призвать первого героя, собрать партию, наладить дороги между поселениями, спрятать тайные знания, открыть порталы и защитить сокровищницу данных. Финалом станет разворачивание доски квестов для таверны приключенцев.

## Быстрый старт

```bash
kind create cluster --config cluster/kind-config.yaml --name k8s-dnd
kubectl apply -f manifests/00-namespace.yaml
kubectl apply -f manifests/01-pod/quest-scroll-pod.yaml
kubectl get pods -n adventure
```

## Сквозное учебное приложение

В репозитории есть приложение `quest-board`:

- показывает название королевства и текущий квест;
- умеет читать настройку из `ConfigMap`;
- умеет видеть, есть ли секретный токен;
- пишет заметки партии в каталог `/data`, чтобы тема хранения была видна не только в теории.
